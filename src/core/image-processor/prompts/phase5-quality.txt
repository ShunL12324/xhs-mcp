## 配图数量

## 检查项目

请逐一检查每张配图：

1. **文字正确性**：是否有错别字、乱码、文字被截断
2. **文字可读性**：字号是否适合手机阅读，对比度是否足够
3. **截图清晰度**：截图是否清晰，关键内容是否可见
4. **标注准确性**：圆圈/箭头是否指向正确位置
5. **整体美观度**：排版是否协调，配色是否统一
6. **内容完整性**：是否覆盖了笔记中的所有重要内容
7. **顺序正确性**：步骤序号是否按正确顺序排列（如 Step 1、Step 2、Step 3...）

## 返回 JSON 格式

正常情况（质量通过，顺序正确）：

```json
{
  "overallPassed": true,
  "issues": [],
  "summary": "所有配图质量良好，文字清晰无误，排版美观。"
}
```

如果有严重质量问题（需要返工）：

```json
{
  "overallPassed": false,
  "issues": [
    "第 2 张图的标题有错别字：'配制' 应为 '配置'",
    "第 3 张图的文字太小，手机上难以阅读"
  ],
  "summary": "发现 2 个问题需要修复，主要是文字错误。"
}
```

**如果只是顺序问题（不需要返工，只需调整输出顺序）**：

```json
{
  "overallPassed": true,
  "issues": [],
  "summary": "配图质量良好，但顺序需要调整。",
  "suggestedOrder": [0, 2, 1, 3, 4, 5, 6]
}
```

注意：`suggestedOrder` 是建议的正确顺序，使用 0-based 索引。例如 `[0, 2, 1, 3]` 表示原来的第 3 张图应该放到第 2 位，原来的第 2 张图应该放到第 3 位。

## 检查原则

1. **文字正确性是最重要的**：任何错别字、乱码都应该标记为不通过
2. **内容完整性**：确保笔记中的每个重要步骤/要点都有对应的配图
3. **宽容小瑕疵**：排版上的小瑕疵可以通过，但影响阅读的问题需要标记
4. **实用性优先**：关注用户能否通过配图理解笔记内容
5. **顺序问题单独处理**：顺序问题不影响 overallPassed，只需在 suggestedOrder 中给出正确顺序
